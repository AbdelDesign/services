Regras para execução do prompt – “Site Abdel Design”
1. Base do projeto

O site deve ser feito em React + Tailwind CSS + GSAP.

Estrutura SPA (Single Page Application) — todas as seções no mesmo arquivo.

Código limpo, sem dependências inúteis e com componentes organizados.

Cada seção é um componente independente (Hero, About, Services, etc.).

Usar Tailwind para estilização com utilitários bem definidos.

2. Design e estilo

Estética sólida, madura e profissional.

Proibido: aparência de IA, gradientes artificiais, estilos “leves”, infantis ou com excesso de brilhos.

Cores: base sóbria (preto, branco, cinza, off-white) com accent elegante (ex: dourado, azul petróleo, ou ciano escuro).

Tipografia: moderna e limpa — Inter, Outfit ou Satoshi.

Espaçamento: usar padding e margin amplos, nunca apertado.

Sombras: sutis e difusas para criar profundidade, sem exageros.

Bordas: radius entre 20px e 30px.

Transições: rápidas e suaves (0.2s–0.35s).

Hover states: sofisticados (leve elevação, mudança sutil de cor ou brilho).

Sem emojis.

3. Responsividade

Dois layouts distintos: um para desktop e outro para mobile/tablet.

Mudança de layout feita via media queries, sem páginas separadas.

Breakpoints recomendados:

min-width: 1024px → layout desktop

max-width: 1023px → layout mobile

Fundos e composições mudam entre layouts, mantendo coerência visual.

4. GSAP – Regras de animação

Usar somente GSAP para animações, evitando transições CSS para os elementos principais.
Animações devem ser leves, elegantes e precisas, com fluidez natural.

Plugins obrigatórios:

ScrollTrigger — ativa animações on-scroll.

SplitText — textos dinâmicos no Hero.

ScrollSmoother — rolagem fluida e parallax.

TextPlugin — efeitos de escrita suave.

MotionPathPlugin — microinterações de ícones.

Timeline — sequências encadeadas.

Animações por seção:

Hero: SplitText com stagger: 0.1, ease: "power2.out", y: 50 → 0, opacity: 0 → 1.

Menu: ScrollTrigger reduzindo altura e aplicando blur no scroll.

Sobre: fade + scale-in (ScrollTrigger).

Serviços: Timeline com autoHeight, ease: "power1.out", expansão de card suave.

Transições gerais: ScrollTrigger + stagger para fade e slide entre seções.

5. Interatividade e UX

Feedback visual em todo clique, hover e focus.

Botões com GSAP.to() (ease: "elastic.out(1, 0.4)" no hover).

Carrossel com scroll-snap + GSAP control.

Carrinho aparece apenas com serviços selecionados (entrada fromTo com scale e slide-up).

WhatsApp abre com mensagem formatada (URL encoded).

Experiência fluida e responsiva em qualquer tela.

6. Estrutura de código

Componentes nomeados claramente (Hero.jsx, Services.jsx, etc.).

Arquivo único gsapAnimations.js para centralizar animações.

Animações inicializadas via useLayoutEffect + useRef.

Estados globais com Context API (para carrinho e serviços).

Classes Tailwind bem definidas, sem CSS inline.

7. Performance

Imagens otimizadas com lazy loading.

Animações pausam fora da viewport.

Compressão e minificação automáticas.

Evitar animações contínuas e pesadas.

8. Identidade Abdel Design

Inserir assinatura discreta “Feito por Abdel Design” no footer.

Todo o projeto deve refletir uma marca de alto padrão, com design maduro, técnico e autêntico — nada com aparência de IA.
